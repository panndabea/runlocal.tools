Open image tools

Simple tools. Run locally in your browser.

A universal toolbox for images, video, data, 3D, and more — without uploading your files to random websites.

Most online utility tools are cluttered, slow, and crowded with ads. A straightforward task gets buried under distractions and misleading buttons.

They also ask you to upload private files to unknown servers. In many cases, you cannot tell what happens next, where your data goes, or how long it is kept.

This should not be necessary for simple work. runlocal.tools runs entirely in your browser, requires no uploads, stays open source, and is designed for tools that can be shared and reused.

A local-first toolbox for the web

Tools are exposed as simple URLs and built to be easy to share, inspect, and combine.

You can open focused tools directly by URL, such as /images/resize or /csv/to-json. Each tool has one clear purpose and runs locally in the browser.

Because there is no required server-side processing for these tasks, links are straightforward to share. Send the URL, open it, use it, and continue.

Tools are also designed to be composable. The output of one tool can become the input to another, inspired by Unix pipes but adapted for browser workflows.

Model

  • Simple routes for specific tasks
  • Processing happens on your device
  • Links are shareable and reusable
  • Tools compose into larger workflows

Why this matters

Local-first behavior should be normal for common file work. It is a practical design choice, not a marketing claim.

Privacy should be normal

Simple file tasks should not require trust in unknown services or terms that users never intended to accept.

Control should stay with users

When processing runs in the browser, people remain in charge of their own files, timing, and workflow.

The browser is already powerful enough

Modern web capabilities make local processing practical for many everyday image, data, and media transformations.

Public software can still be usable

Open tools can be thoughtfully designed, reliable, and calm to use without ad-heavy interfaces or lock-in patterns.

Open source by design

Credibility comes from transparency. Every part of the project is intended to be inspectable and reusable.

  • The project is fully open source.
  • Tools can be inspected, adapted, and reused in other contexts.
  • Teams can self-host the toolbox when needed.
  • The project is built as digital commons, not a closed platform.

Roadmap

The direction is practical: more useful tools, stronger shared infrastructure, and better composition.

  • Expand tool coverage across images, video, data, 3D, and CAD.
  • Develop a shared local processing engine for consistency.
  • Improve composable workflows between tools.
  • Explore optional local AI integrations later, when useful.

About

runlocal.tools is a long-term open web project focused on trustworthy utility software. It prioritizes privacy, inspectability, and clear interaction design for everyday tasks.
