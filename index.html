A local-first toolbox for the web

runlocal.tools is a growing collection of practical tools for everyday file and media tasks.

Each tool has a simple, stable URL:

  • /images/resize
  • /images/convert
  • /csv/to-json
  • /json/to-csv

You open a link, drop a file, and the work happens locally — on your device.

No server round-trips. No file transfers. No waiting for someone else’s infrastructure.

Just the tool, running where your data already is.

Local execution model

  • Simple URLs for focused tasks
  • Browser-native processing
  • No required uploads
  • Reliable and shareable workflows

Share tools, not files

Because each tool is accessible via a simple URL, it can be shared easily.

Instead of sending files to services, you send the tool itself.

“Use this to resize your image.”

“Convert your CSV here.”

The person on the other end runs the same tool — locally, in their own browser.

No accounts. No uploads. No friction.

Composable by design

runlocal.tools is not just a collection of isolated tools.

It is designed to grow into a system where tools can be combined.

The output of one tool can become the input of another. Small, focused actions can be chained into larger workflows.

Inspired by Unix pipelines — but adapted to the browser.

Simple building blocks. Clear inputs and outputs. Composable by design.

Why this matters

Everyday file tasks should not require blind trust.

Uploading personal files to unknown services has quietly become normal. But for most tasks, it is unnecessary.

Modern browsers are powerful enough to handle many of these operations locally.

runlocal.tools is built on a simple idea:

Users should stay in control of their data.

  • No forced uploads
  • No hidden processing
  • No dependency on opaque services

Just tools that run where your data already lives.

Open source, by default

runlocal.tools is built as an open project.

  • Source code is publicly available
  • Tools can be inspected and verified
  • Components can be reused and extended
  • The platform can be self-hosted

This is not a closed service. It is a set of tools meant to be shared, improved, and built upon.

Built as digital commons

runlocal.tools is intended as public digital infrastructure.

A place where common tools are:

  • freely accessible
  • openly developed
  • usable without friction
  • independent from centralized services

Simple utilities should not be locked behind ads, tracking, or opaque systems. They should be part of the commons.

What’s next

The initial focus is on a small set of reliable tools:

  • image resize, convert, compress
  • CSV ↔ JSON conversion

From there:

  • expand into more tools across images, video, and data
  • build a shared local processing layer
  • enable composable workflows between tools
  • explore optional local AI-assisted processing

The long-term goal is simple:

A universal, open toolbox for the web — running locally.

About

runlocal.tools is a public-interest project for practical browser-native tools, built with a local-first mindset and an open development model.

Privacy

For supported tools, files are processed locally in your browser on your device and are not uploaded to project-operated servers for processing.
